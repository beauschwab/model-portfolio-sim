## Anti-Patterns Verdict

**LLM assessment:** Not a visual dumpster fire. The dark Binance theme is cohesive, the typography is restrained, numbers are tabular. The real problem is *interaction inefficiency*, not aesthetics. Two slop tells: (1) the Risk Desk 2×3 identical stat-card grid, and (2) oversized empty chart cards with centered "no data — run X" gray text. Morning Sheet has a full empty viewport below the fold.

**Deterministic scan:** detect.mjs over apps/web/src → clean (exit 0, []). No markup-level anti-patterns.

## Overall Impression
The engine is fast and the domain language is impeccable, but the UI gates everything behind slow, serial, manual runs with almost no feedback — which is exactly the "inefficiency" complaint. Positions proves the team can build a dense, efficient surface; Risk Desk and the run-orchestration model don't live up to it. Biggest opportunity: make data populate by default (or in parallel, with progress) so the tool feels live.

## What's Working
1. **Positions** — dense KRD heatmap, expandable Assets→book→position tree, view toggle, editable balances via slider popovers. This is the high-water mark.
2. **Domain fluency** — speaks trader natively; the constraint ledger with inline limits + headroom bars is excellent.
3. **Honesty surfacing** — "figures indicative / Risk Desk runs authoritative", synthetic-book footer, InfoPop method notes.

## Priority Issues
- **[P1] Long runs give almost no feedback.** 15-30s+ with a disabled "running…" label and three large empty card voids. → harden (loading/error states), animate (progress).
- **[P1] Manual + serial interaction model.** Three mutually-blocking run buttons; Positions fetches 6 books sequentially instead of Promise.all. Breaks the "slider speed" promise. → optimize.
- **[P2] Risk Desk wastes the viewport.** 2×3 identical stat grid + oversized empty chart cards push real content below the fold. → layout.
- **[P2] Errors via alert().** Native modal for job failure; no inline error or retry. → clarify + harden.
- **[P2] 381 rows unvirtualized.** Positions renders every row in the DOM; edit/scroll perf risk on the densest surface. → optimize.
- **[P3] Color-only encoding.** KRD heatmap + up/down rely on green/red alone; add sign/arrow for color-vision deficiency. → colorize/audit.
